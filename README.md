# 本地去水印处理系统 (Wm-Remover)

本仓库提供一套完整的本地化去水印处理系统，支持通过 Web 界面上传并自动处理图像水印及背景修复。系统采用本地化引擎运行，无需外部网络调用。

## 🏗️ 系统架构

系统采用 **Node.js + Python** 的分层处理架构：

### 1. 后端服务 (Node.js - 5004 端口)
- **核心逻辑**: 位于 `watermark-remover/backend`。
- **主要职责**: 处理客户端图像上传、任务队列管理、处理结果的分发与存储。
- **网络状态**: 监听 `0.0.0.0:5004`，支持 Web 访问。

### 2. 处理引擎 (Python - 5005 端口)
- **核心逻辑**: 位于 `ai-engine/` 目录。
- **技术原理**: 基于图像修补 (Inpainting) 技术的本地化实现，专用于背景像素重构。
- **主要职责**: 执行高精度的像素级修复任务。
- **网络状态**: 监听 `127.0.0.1:5005`，仅供后端内网调用以确保性能和隐私。

## 📂 目录结构

- `ai-engine/`: 处理引擎核心服务，包含模型加载及推理逻辑。
- `watermark-remover/`: 后端 API 服务及 Web 前端静态页面。
- `scripts/`: 系统维护及自动化运维脚本。

## 🚀 运行说明

1. **启动引擎**: 运行 `ai-engine/venv/bin/python3 lama_service.py` 开启底层处理端口。
2. **启动后端**: 运行 `watermark-remover/backend/dist/index.js` 开启 Web 访问入口。

---
*注：本系统所有处理逻辑均在本地服务器完成，不涉及第三方接口调用。*
